## Background

### Context

The base Tier 1 roster is 8 personas (medium effort). The user has indicated openness to 7-11 personas depending on findings. This investigation evaluates whether adding a 9th or 10th persona (the winners from I-R1 and I-R2) provides sufficient marginal value to justify the additional synthesis complexity and token cost.

### Key Dimensions

**Synthesis complexity burden:**
- 8 personas: manageable clustering and synthesis time
- 9 personas: additional clustering overhead, increased disambiguation requirements
- 10+ personas: multiplicative synthesis complexity, potential for diminishing returns

**Token cost implications:**
- Each additional persona adds ~30-60 KB of Phase 2B input content
- Synthesis pass must reconcile all perspectives, creating downstream context pressure
- Risk of "too many voices" making Phase 4 synthesis harder to resolve

### Research Questions

1. **RQ11a Marginal territory:** What % of the promotion candidate's questions cover themes NOT already present in the Tier 1 aggregate? (Target: >30% new territory to justify 9th addition)

2. **RQ11b Dimension balance:** Does adding the 9th persona improve balance across dimensions or add to an already-strong dimension?

3. **RQ11c Synthesis complexity:** With 9-10 personas at target volume, what is the total question count and clustering structure? Is Phase 4 synthesis still manageable?

4. **RQ11d Diminishing returns:** Compare dimension coverage curves:
   - 8 personas: baseline coverage across all dimensions
   - 9 personas: incremental coverage added
   - 10 personas: further incremental coverage added
   - Is the 8→9 jump larger than 9→10? (Expected: yes)

5. **RQ11e Question-mode cluster representation:** Does the 9th/10th persona introduce a question mode or cluster not already represented? Or do they primarily amplify existing clusters?

## Investigation Task

### I-R4: Marginal Value of 9th and 10th Persona

**Purpose:** Determine whether adding a 9th or 10th persona to the Tier 1 set provides sufficient marginal value to justify synthesis complexity and token cost increase.

**Key question:** Given the Tier 1 set's coverage, how much unique territory does a 9th and 10th persona add? Is the marginal return worth the additional synthesis burden?

**PREREQUISITE:** I-R1 and I-R2 must complete first. This investigation uses their recommendations to determine which persona(s) to analyze.

**Files to analyze:**

*Tier 1 q15-20 baseline (24 files, ~175 KB):*
- All 8 Tier 1 personas across 3 topics: space-party, tool-library, habit-tracker
  - Questioner, Analyst, Devil's Advocate, Provocateur (analytical cluster)
  - Visionary, Pragmatist, Storyteller, Appreciative Inquirer (aspirational/human-centered cluster)

*Promotion candidate #1 q15-20 from I-R1 winner (3 files, ~22-26 KB):*
- [If Audience Advocate promoted: AA q15-20]
- [If Empath promoted: Empath q15-20]

*Promotion candidate #2 q15-20 from I-R2 winner (3 files, ~20-31 KB):*
- [If Systems Thinker promoted: ST q15-20]
- [If First Principles Thinker promoted: FPT q15-20]

*Total scope: 30 files, ~218-252 KB (depends on I-R1 and I-R2 recommendations)*

**Extended data available:** All 4 promotion candidates have data at q05-10 and q10-15 volumes (generated by I-DG1/I-DG2), enabling marginal value assessment at the volume level selected by Decision Gate 1, not just at q15-20.

**Comparison criteria:**

1. **Marginal territory assessment:**
   - For the 9th persona: Calculate % of questions that cover themes, dimensions, or question modes not present in the Tier 1 aggregate
   - Threshold: >30% new territory to justify addition (i.e., candidate must bring substantive new perspective)
   - For the 10th persona: Repeat for second candidate
   - Compare: Is 9th persona's marginal value higher than 10th's? (Expected: yes)

2. **Dimension balance impact:**
   - Map both Tier 1 and Tier 1 + 9th persona across all dimensions (Strategic, Tactical, Creative, Analytical, Human-centered, Systems, Provocative)
   - Identify: Does 9th persona fill an underrepresented dimension or amplify an already-strong one?
   - For each scenario (keep 8, expand to 9, expand to 10), calculate dimension balance (standard deviation across % per dimension)
   - Better balance = lower SD across dimensions

3. **Synthesis complexity assessment:**
   - 8 personas: Total question count, clustering structure
   - 9 personas: Incremental questions added, new clusters introduced or existing clusters grown?
   - 10 personas: Further incremental analysis
   - For Phase 4 synthesis, estimate effort required to reconcile perspectives (manageable / moderate difficulty / high complexity)

4. **Diminishing returns curve:**
   - Create comparison table:
     - Dimension coverage: % of each dimension represented at 8-persona vs 9-persona vs 10-persona
     - New territory: % of candidate's questions that are new at 9-persona vs 10-persona (calculating as "new to tier 1")
     - Question clustering: # of distinct themes/clusters at each level
   - Visual: Does diminishing returns curve show significant 8→9 jump but smaller 9→10 jump?

5. **Question-mode cluster representation:**
   - Review cluster themes/modes for Tier 1, Tier 1+9th, and Tier 1+10th
   - Identify: Are the candidates introducing fundamentally new question modes (e.g., a cluster no other persona generates)?
   - Or are they amplifying existing clusters with more variations?
   - High value = new mode. Lower value = variations on existing themes.

6. **User benefit assessment:**
   - Consider the user's stated preference for "open scope" (willing to add personas if value clear)
   - Weigh findings: Does the user get proportionally better brainstorm from 9 vs 8? From 10 vs 9?
   - Consider: If marginal value is >15% at 9 but <8% at 10, recommend stopping at 9

**Expected output:**

- Tier 1 baseline coverage profile
- Marginal value assessment for 9th persona (with recommendation: Add / Don't add)
- Marginal value assessment for 10th persona (with recommendation: Add / Don't add)
- Synthesis complexity and diminishing returns analysis
- Final roster size recommendation with rationale
- Effort-level guidance: "At medium effort, recommend X personas. At high effort, could expand to Y."

**Priority:** Medium — depends on I-R1 and I-R2 completion; cannot start until those investigations complete. Informs final roster size decision before high-effort specialist selection.
