# Simplifier Viability Assessment: mobile-app Topic

**Date:** 2026-02-24
**Purpose:** Determine whether The Simplifier's conditional Tier 3 retention is justified by assessing whether it produces genuinely unique questions on its strongest trigger topic (mobile-app: "strip it down to its essential core").

**Method:** Cross-reference all Simplifier questions (27 total across two runs: q08-12 = 12, q10-15 = 15) against all other personas' output on the same topic. Identify genuinely unique territory, then assess whether that territory justifies the orchestrator overhead of conditional inclusion.

### Questions marked "Partially Unique" — detailed assessment

**Stealth removal / quiet disappearance experiment (Q2 run1, Q4 run2):**
- Simplifier Q2 (run1): "What would happen if you tracked actual usage and removed the bottom 80% of features with zero announcement?"
- Simplifier Q4 (run2): "What if the 'clean relaunch' was simply deleting the unused code and seeing if anyone notices?"
- Nearest competitor: Questioner (q10-15 Q6) proposes testing feature removal before committing — "Could we ship a version with one feature temporarily disabled and see who complains?" — but frames this as an experiment, not as stealth deletion.
- The Simplifier's framing is more radical: remove silently and wait for complaints. The Questioner suggests controlled testing. The Provocateur (q05-08 Q2) proposes a "controlled demolition" variant but takes it to an absurd extreme.
- Assessment: The Simplifier's "stealth removal" framing is **partially covered** by the Questioner's testing-based approach and the Provocateur's controlled demolition. The specific mechanism (remove silently, measure complaints) has a narrow unique angle, but it is a methodological suggestion rather than a generative brainstorming question.

**Pure subtraction without replacement (Q5 run1):**
- Simplifier: "What if the relaunch was purely subtractive, with the new app having literally fewer capabilities?"
- Nearest competitor: Constraint Flipper (q08-12 Q4): "What if feature removal creates value rather than destroying it?" frames the same territory positively.
- Assessment: The Constraint Flipper covers the same territory from a different angle. The Simplifier's specific emphasis on "no replacement, just removal" is a narrow unique angle, but is a one-line reframing rather than a generative question.

**Marketing "now with less" (Q9 run1):**
- Nearest competitor: Constraint Flipper (q05-08 Q2): "What if the radical act of removing 11 features became the most powerful marketing message possible?" and Storyteller (q05-08 Q6) on positioning simplification as innovation.
- Assessment: **Covered.** The Constraint Flipper explores this marketing angle with equal or greater depth, and the Storyteller addresses the narrative framing question more thoroughly.

**Willingness-to-pay as simplification filter (Q2 run2):**
- Simplifier: "What if we asked users which features they'd pay to keep rather than which features they like?"
- Nearest competitor: Accountant (q08-12 Q4): Churn cost versus feature elimination cost, which touches on revenue valuation of features. Analyst (q08-12 Q1): "How would we design an experimental approach to distinguish features that users say they want from features they actually use?"
- Assessment: The Analyst and Accountant address the stated/revealed preference gap and feature valuation, but neither proposes the specific mechanism of "willingness to pay to keep." This is a **genuinely useful methodological suggestion** not directly covered elsewhere. However, it is a single question, and it is a research methodology suggestion rather than a generative brainstorming question.

**Change the promise, not the features (Q7 run2):**
- Simplifier: "What if the 'feature-bloat' problem is actually a 'user expectations' problem — and we solve it by changing what we call the product?"
- Nearest competitor: Questioner (q15-20 Q8): "How would users describe this app to a friend in one sentence today versus ideally?" and Politician (q10-15 Q14): "The relaunch narrative and stakeholder signaling."
- Assessment: The Simplifier's reframing — keep features but change the stated promise — is partially unique. The Questioner and Politician approach the narrative angle from different directions. The Simplifier's specific mechanism (keep more features, simplify the promise) is a narrow but genuinely different strategy from the majority "keep fewer features" direction. However, it contradicts the Simplifier's own core mode (subtraction), making it an anomalous output.

**Single-notification hypothesis (Q14 run2):**
- Simplifier: "What if we cut all notification channels except one?"
- Nearest competitor: Empath (q10-15 Q3): Notification emotional toll. Audience Advocate (q08-12 Q9): Notification philosophy respecting user attention.
- Assessment: Other personas address notification overload as a problem, but none propose the specific solution of reducing to exactly one notification type. This is a **concrete operational suggestion** rather than a brainstorming question. It is actionable but convergent.

**Feature divestiture to specialists (Q13 run2):**

## Unique Residual

After thorough cross-referencing against all 18 other persona outputs on the mobile-app topic:

**Genuinely unique questions:** 0 out of 27.

No Simplifier question covers territory that is entirely unaddressed by other personas. Every line of inquiry the Simplifier opens on its strong-trigger topic has at least one near-equivalent from another persona.

**Partially unique questions:** 7 out of 27 (~26%).

These questions have a distinctive angle, framing, or mechanism not precisely replicated by other personas:
1. Stealth removal experiment (Q2 run1, Q4 run2) — narrow methodological angle
2. Willingness-to-pay-to-keep filter (Q2 run2) — useful research methodology suggestion
3. Pure subtraction without replacement (Q5 run1) — one-line reframing
4. Change the promise, not the features (Q7 run2) — anomalous for the persona's mode
5. Navigation hierarchy as concrete cut criterion (Q5 run2) — operational heuristic
6. Feature divestiture to specialists (Q13 run2) — business development angle
7. Single notification channel (Q14 run2) — concrete operational suggestion

**Character of the partially unique residual:**
- 6 of 7 are **convergent** — they propose specific operational actions or evaluation methods, not open-ended exploration directions.
- 1 of 7 (Q7 run2: change the promise) is arguably **divergent** but contradicts the persona's core cognitive mode, suggesting it is an anomalous output rather than a reliable persona contribution.
- None of the 7 represent a thematic territory that would be noticeably absent from Phase 2B synthesis without the Simplifier. They are incremental refinements of territories already explored by other personas.

**Covered questions:** 20 out of 27 (~74%).

The overwhelming majority of the Simplifier's output on its strongest trigger topic duplicates territory already covered by:
- **Analyst** (feature-level metrics, data-driven prioritization, unit economics of removal)
- **First Principles Thinker** (minimum viable product, irreducible core, zero-based feature budgeting)
- **Questioner** (testing before committing, feature removal experiments, redefining success)
- **Constraint Flipper** (constraints as advantages, marketing less as differentiation, runway as focus engine)
- **Devil's Advocate** (simplification backfiring, passionate minority risks, technical debt from removal)
- **Systems Thinker** (feature interdependencies, simplification-creep prevention, cascading effects)
- **Visionary** (zero-feature product, no-UI experience, one-button design)

The first five of these personas collectively saturate the simplification/de-scoping territory that defines the Simplifier's entire cognitive mode.

## Recommendation: Demote to Non-Question Generating

**Verdict: Demote The Simplifier from conditional Tier 3 to Non-Question Generating (excluded from Phase 2B entirely).**

### Rationale

1. **Zero unique territory on its strongest trigger topic.** The mobile-app topic was specifically designed as the Simplifier's ideal use case — explicit "strip it down" framing, 15 features to cut to 3-4, an 8-month runway adding urgency. If the Simplifier cannot produce genuinely unique questions even here, there is no plausible topic where it would add value that other personas miss.

2. **The simplification/de-scoping territory is already saturated.** Five personas — Analyst, First Principles Thinker, Questioner, Constraint Flipper, and Devil's Advocate — collectively cover the entire simplification space with greater depth and more diverse framing than the Simplifier. The Constraint Flipper in particular covers "simplification as strategic advantage" territory with a generative orientation that the Simplifier's convergent mode cannot match. The First Principles Thinker covers "irreducible core" territory with more philosophical depth. The Analyst covers "which features to cut based on evidence" territory with more analytical rigor.

3. **The partially unique residual is operationally convergent, not generatively useful.** The 7 partially unique questions are specific methodological suggestions (stealth removal, willingness-to-pay filter, navigation heuristic) or operational proposals (sell features to specialists, single notification). These are more appropriate as Phase 3 action items than Phase 2B brainstorming questions. Their convergent character confirms the R6 finding that the Simplifier's cognitive mode is evaluative, not generative.

4. **The conditional retention clause adds orchestrator complexity for zero demonstrated benefit.** Retaining the Simplifier as a conditional option requires the orchestrator to: (a) detect simplification trigger language, (b) select and invoke the persona, (c) incorporate its output into synthesis. On the mobile-app topic — the conditional clause's best-case scenario — the synthesis output would be functionally identical with or without the Simplifier's questions. The orchestrator overhead is not justified.

5. **The Constraint Flipper is the superior simplification persona.** Where the Simplifier asks "What if we removed X?" (convergent), the Constraint Flipper asks "What if constraint X is actually an advantage?" (generative). Both address simplification territory, but the Constraint Flipper opens new lines of inquiry while the Simplifier narrows to evaluation. On the mobile-app topic, the Constraint Flipper (30 questions across 3 runs) produces more distinctive, generative coverage of simplification territory than the Simplifier (27 questions across 2 runs). The Constraint Flipper is already Tier 2 and does not require conditional activation logic.

### Implementation

- Remove The Simplifier from the Phase 2B persona selection guide entirely.
- Reclassify as "Non-Question Generating" alongside other personas excluded from Phase 2B (e.g., Pragmatist).
- No changes needed to Phase 2C synthesis logic, as the conditional clause was never implemented in production.
- Consider noting in the persona library that the Simplifier's cognitive mode remains available for Phase 3 evaluation contexts, where convergent "should we remove this?" questions are directly productive.
